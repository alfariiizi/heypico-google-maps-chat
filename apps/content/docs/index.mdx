# HeyPico Maps - LLM + Google Maps Integration

A **production-ready** system that integrates a **Local LLM** with **Google Maps API**, allowing users to search for places and get directions through natural language queries.

**GitHub Repository:** [https://github.com/alfariiizi/heypico-google-maps-chat](https://github.com/alfariiizi/heypico-google-maps-chat)

##  What is This?

This project demonstrates how to build a modern AI-powered location service by combining:

- **Local LLM** (Ollama) for natural language understanding
- **Google Maps API** for location data and directions
- **Backend API** (Hono + TypeScript) as a secure bridge
- **Open WebUI** for the chat interface

##  Architecture

```
User: "Find Italian restaurants in New York"
    ↓
Open WebUI (localhost:3000)
    ↓
Local LLM - Ollama (llama3.2/mistral)
    ↓ [Function Call]
Backend API - Hono (localhost:8432)
    ↓
Google Maps API
    ↓
Response with maps & links
```

##  What's Included

-  **Docker Setup**: Ollama + Open WebUI (`docker-compose.yml`)
-  **Backend API**: Production-ready Hono + TypeScript API
-  **Google Maps Integration**: Secure API wrapper with all major endpoints
-  **Open WebUI Functions**: Python functions for LLM integration
-  **Comprehensive Documentation**: This documentation site!

##  What You'll Learn

This project demonstrates:

- Modern TypeScript development with strict mode
- REST API design with Hono framework
- Third-party API integration (Google Maps)
- LLM function calling patterns
- Security best practices (API key protection, rate limiting)
- Error handling & validation with Zod
- Clean code architecture
- Docker containerization
